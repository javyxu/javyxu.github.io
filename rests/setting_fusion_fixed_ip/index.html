在Fusion下设置固定IP

Posted on 2018-11-08 Edited on 2021-02-21 In Linux Views:

最近在Mac上用Fusion跑Linux，主要是用来做server端的开发，但是在开发过程中发现，虚拟机只要重新启动，IP地址便会跟着发生变化，这很是让人烦恼。

后来想着如何能让虚机IP固定下来呢，Google良久，终于有可固定的解决方式，下面请看详细解决步骤。

我本人的环境是，macOS版本是10.14.1，VMware Fusion的版本是Professional Version 11.0.0 (10120384)。

1. 查询虚拟机的MAC地址

话不多说，直接上图

设置-网络适配器

设置-网络是配置-MAC地址

2. 修改dhcpd.conf

dhcpd.conf位于目录/Library/Preferences/VMware Fusion/vmnet8。

用你最喜欢的文本编辑器打开/Library/Preferences/VMware Fusion/vmnet8/dhcpd.conf，我这里用vim，需要用管理员权限sudo,如下所示：
sudo vim /Library/Preferences/VMware\ Fusion/vmnet8/dhcpd.conf
看到的内容大概是这样：

** 有三行内容值得注意：**

  • 第1行，Ubuntu 是虚拟机的名字，看下图，注意要拼写要完全一致。

  • 第2行，00:0C:29:F1:67:98是上一步获取的MAC地址。

  • 第3行，192.168.110.130是要设置的固定IP地址，注意要在虚拟机的IP网段，一般情况下就用上次虚拟机运行时的动态IP就可以了。

3. 重启VMWare Fusion

必须重启VMWare Fusion才能使上边的设置生效。

4. 启动虚拟机

此时在启动虚拟机，你会发现虚拟机的地址不会再变来变去了，永远是你在步骤2设置的固定IP。
